# my-kb

這個專案是一組給 AI Agent 使用的知識庫與交接規則。
它的目的不是提供執行中的應用程式，而是讓不同 Agent 在同一個 repo 裡能夠一致地工作、交接與追蹤進度。

## 專案用途

- 集中管理 Agent 共用規則。
- 提供不同角色的工作準則。
- 定義交接格式，讓任務可以跨 Agent 延續。
- 作為版本控管時的狀態紀錄基礎。

## 目前功能

- 共用規則文件：`AGENTS.md`
- 角色規則：
  - `.agents/planner.md`
  - `.agents/research.md`
  - `.agents/implementation.md`
  - `.agents/review.md`
- 交接模板：`.agents/handoff.md`
- 使用說明：`.agents/README.md`、`.agents/usage.md`

## 啟動方式

這個專案沒有可執行的服務或前端應用，因此不需要啟動伺服器。
要使用它，直接打開文件並依流程工作即可：

1. 先讀 `AGENTS.md`
2. 再讀對應角色規則
3. 任務完成後填寫交接內容

## 部署方式

這個專案不需要傳統部署。
若要在團隊中使用，只要把 repo 推到 GitHub，之後所有協作者或 AI Agent 都能透過版本控管取得最新規則與進度。

## 環境變數
