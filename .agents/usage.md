# Agent 使用手冊

這份手冊用來讓新 Agent 快速理解這個 repo 的工作方式、接手順序與交接標準。
如果你是第一次進來，請先讀共用規則，再讀你的角色規則，最後看最近一筆進度紀錄。

## 1. 開始前

1. 讀 `[AGENTS.md](../AGENTS.md)`，確認全域規則。
2. 讀 `.agents/README.md`，確認目前可用的角色與閱讀順序。
3. 讀自己的角色規則檔。
4. 讀上一輪留下的 `Handoff` 與 `docs/PROGRESS.md`。
5. 確認當前目標、已完成內容、未完成內容與風險。

## 2. 工作中

- 只處理當前角色應負責的部分。
- 維持變更範圍小而清楚。
- 修改前先列出本次要變更的檔案。
- 修改後要整理改動內容、影響範圍與下一步建議。
- 如果發現任務方向需要調整，先留下原因，再交回適合的角色。
- 如果需要查證或補背景，優先把問題寫清楚，再交給 Research。
- 如果需要回收任務切換或重拆步驟，交回 Planner。

## 3. 結束時

- 寫清楚做了什麼。
- 寫清楚還剩什麼。
- 寫清楚驗證結果、風險與假設。
- 若有修改檔案，列出實際檔名。
- 若任務完成，更新 `docs/PROGRESS.md`。
- 若需要交接，使用 `.agents/handoff.md` 的格式。

## 4. 建議流程

```md
Planner -> Research -> Implementation -> Review
```

這是預設順序，不是硬性限制。實際執行時可依任務性質回到前一個角色，但交接格式應保持一致。

## 5. 交接標準

每次交接至少要包含：

- Current goal
- Done
- Changed files
- Remaining
- Risks / assumptions
- Verification

如果某一項沒有內容，請明確寫 `none`，不要留空。

## 6. 常見錯誤

- 只寫結果，不寫過程與風險。
- 修改了檔案卻沒更新進度紀錄。
- 交接內容太抽象，下一個 Agent 無法直接接手。
- 一次改太多檔案，讓變更難以追蹤。

## 7. 工作原則

- 先確認，再修改。
- 先交代，再推進。
- 先讓下一個 Agent 看得懂，再追求一次做完。
