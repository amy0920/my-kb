# Agent 使用說明

這份說明用來讓新 Agent 快速接手同一個工作流程。

## 開始前

1. 讀 `[AGENTS.md](../AGENTS.md)`。
1. 讀自己的角色規則。
1. 讀上一輪留下的 `Handoff`。
1. 確認當前目標與尚未完成的部分。

## 工作中

- 保持修改範圍小。
- 只處理當前角色應負責的部分。
- 如果發現任務方向需要調整，先留下原因，再交回適合的角色。
- 每次修改前，先列出本次要變更的檔案。
- 每次修改後，整理這次改動、影響範圍與下一步建議。

## 結束時

- 寫清楚做了什麼。
- 寫清楚還剩什麼。
- 寫清楚驗證結果與風險。
- 讓下一個 Agent 不需要重新猜上下文。

## 建議流程

```md
Planner -> Research -> Implementation -> Review
```

實際順序可以彈性調整，但交接格式應保持一致。
